# 策略 A：持续积累型

## 适用场景

长期、持续积累的知识领域，内容不失效或很少失效，跟随某个主体（人、项目、组织）的成长持续演进。

**典型场景**：育儿知识库、个人健康管理、长期技能修炼记录

**核心特征**：内容会持续增长很多年，但每个阶段只关注当下该做的事。按时间轴组织，主题标签跨阶段串联。

## 六维度定义

| 维度 | 定义 |
|------|------|
| 生命周期 | 持续型，不封存。子主题（如年龄段）自然度过 |
| 内容流入 | 持续流入（日志按周期，卡片按需） |
| 核心产出 | 带信息源分级的知识卡片 + 主体成长记录 |
| 谁写什么 | LLM 编译卡片草稿；人写成长记录、审核卡片 |
| 时效处理 | 基本不失效；政策/外部规则类过期归档 |
| 退出机制 | 不退出。子主题标记"已度过"，不删 |

## 目录配置（逻辑名）

```json
{
  "directories": {
    "原始资料": { "maintainer": "manual", "writable": false },
    "观察记录": { "maintainer": "manual", "writable": false },
    "学习资料": { "maintainer": "manual", "writable": false },
    "成长日志": { "maintainer": "human", "writable": false },
    "知识卡片": { "maintainer": "llm_draft_human_review", "writable": true, "draft_dir": "知识卡片/.草稿" },
    "阶段重点": { "maintainer": "human", "writable": false },
    "远期观察": { "maintainer": "observe_only", "writable": false }
  },
  "compile_rules": [
    { "input": "观察记录", "output": "成长日志", "rule": "不编译成通识，按周期归档" },
    { "input": "学习资料", "output": "知识卡片", "rule": "编译成卡片，带 source_level 和阶段标记" },
    { "input": "外部政策", "output": "远期观察", "rule": "编译成观察笔记，标只看不练" }
  ]
}
```

## 编译规则

- **观察记录 → 成长日志**：不编译成通识知识，只按周期（如月度）归档。这是个案记录，不是普适知识。
- **学习资料 → 知识卡片**：LLM 编译成结构化卡片，必须带 `source_level`（[S]/[A]/[B]）和阶段标记（如 `age_range`）。未审核的进草稿区。
- **外部政策 → 远期观察**：编译成观察笔记，标注"只观察不行动"。

## 时效处理

- 知识卡片基本不失效（基础科学知识几十年不变）
- 政策/外部规则类内容过期后移入 `{远期观察}/归档/`
- 成长日志永不归档，是原始记录

## 换季机制

每隔一个阶段（如半年到一年），做一次五分钟换季：

1. 新建下一阶段的 `{阶段重点}/下一阶段.md`（复制模板，填新里程碑）
2. 索引文件的"当前阶段"链接指向新文件
3. 浏览 `{知识卡片}` 里匹配新阶段的卡片
4. 旧阶段文件加标记 `> 已度过`，不删

## Lint 检查项

- 卡片是否标了 source_level
- 卡片的阶段标记是否有效
- 阶段重点是否需要换季（距上次更新超过设定周期）
- 远期观察区是否有内容该转为正式研究
- 成长日志是否连续（有无断月）

## 产出模板

| 模板 | 用途 |
|------|------|
| `templates/knowledge-card.md` | 知识卡片（带信息源分级+实践记录） |
| `templates/growth-journal.md` | 成长日志（按月记录） |
| `templates/inbox-item.md` | 灵感库条目（公共） |
