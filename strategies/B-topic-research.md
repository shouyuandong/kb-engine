# 策略 B：主题研究型

## 适用场景

围绕某个主题进行深度研究，研究期间高频投入，研究完封存。一个主题一个 wiki，不合并。

**典型场景**：AI技术学习（蒸馏、具身智能、推理架构等）、学术论文研究、技术选型调研

**核心特征**：有明确的生命周期终点。核心产出不是研究过程中的笔记，而是封存时的研究结论摘要。wiki 是过程，结论是交付物。

## 六维度定义

| 维度 | 定义 |
|------|------|
| 生命周期 | 封存型，研究完封存 |
| 内容流入 | 一次性灌入（研究期间高频） |
| 核心产出 | 研究结论摘要（wiki 本身是过程，结论是产出） |
| 谁写什么 | LLM 编译全部 wiki 内容；人审核 + 写结论 |
| 时效处理 | 封存时标注日期；旧结论标记"可能过时" |
| 退出机制 | 研究完跑最终编译+lint，产出结论摘要，标记封存日期 |

## 目录配置（逻辑名）

```json
{
  "directories": {
    "原始资料": { "maintainer": "manual", "writable": false },
    "研究笔记": { "maintainer": "llm_draft_human_review", "writable": true, "draft_dir": "研究笔记/.草稿" }
  },
  "compile_rules": [
    { "input": "原始资料", "output": "研究笔记", "rule": "编译成结构化笔记，建立反向链接" }
  ]
}
```

## 生命周期三阶段

| 阶段 | 状态 | 做什么 |
|------|------|--------|
| 活跃期 | 正在研究 | raw 持续灌入，LLM 增量编译，频繁审核 |
| 封存期 | 兴趣转移 | LLM 最终编译+lint，产出研究结论摘要，标记封存日期 |
| 归档期 | 长期不碰 | 保留 wiki 和摘要，raw 可压缩备份。回顾时只读摘要 |

## 编译规则

- **原始资料 → 研究笔记**：LLM 读取原始资料，编译成结构化 markdown 笔记。自动：
  - 写摘要
  - 分类成概念
  - 为每个概念写文章
  - 建立文章间的反向链接
- 增量编译：新资料进来只更新受影响的部分，不全量重编

## 封存流程（核心）

当研究告一段落时，执行封存：

1. LLM 跑一次最终编译 + lint
2. 用 `templates/research-conclusion.md` 生成研究结论摘要
3. 摘要包含：结论、判断验证（哪些被证实/证伪）、未解问题、重启入口、关联其他 wiki
4. 标记封存日期
5. raw 资料可压缩备份

> 没有封存摘要的 wiki 就是死城。封存摘要才是真正的交付物。

## 时效处理

- 封存时标注日期
- 技术类领域半年一个代际，封存超过 6 个月的结论标记"可能过时"
- 归档期的 wiki 回顾时只读摘要，不读全文

## Lint 检查项

- 活跃超过设定周期（如 3 个月）未封存的提示
- 封存摘要是否完整（结论/验证/未解问题/重启入口）
- 研究笔记之间是否有断链
- 原始资料是否有未编译的积压

| 模板 | 用途 |
|------|------|
| `templates/research-conclusion.md` | 研究结论摘要（封存产出） |
| `templates/inbox-item.md` | 灵感库条目（公共） |
