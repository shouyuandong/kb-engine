# 策略 C：时效腐烂型

## 适用场景

内容时效性强，快速过期。核心价值不在具体判断（会腐烂），而在框架本身（长线持续迭代）。

**典型场景**：金融交易（研报、信号、复盘）、行业动态追踪、竞品情报

**核心特征**：判断是短线的会腐烂，框架是长线的持续打磨。过期判断归档不删除，保留作为复盘素材。

## 六维度定义

| 维度 | 定义 |
|------|------|
| 生命周期 | 腐烂型，内容快速过期 |
| 内容流入 | 高频持续流入 |
| 核心产出 | 框架本身（方法论/信号体系），不是具体判断 |
| 谁写什么 | LLM 编译摘要、标记时效；人写复盘、定框架、做判断 |
| 时效处理 | 带日期标记，过期判断自动移入归档区 |
| 退出机制 | 不退出，但判断会失效；框架持续迭代 |

## 目录配置（逻辑名）

```json
{
  "directories": {
    "原始资料": { "maintainer": "manual", "writable": false },
    "框架": { "maintainer": "human", "writable": false },
    "认知": { "maintainer": "llm_draft_human_review", "writable": true, "draft_dir": "认知/.草稿" },
    "归档": { "maintainer": "llm_full", "writable": true }
  },
  "compile_rules": [
    { "input": "原始资料", "output": "认知", "rule": "提取要点+标记时效日期，不保留原文" },
    { "input": "过期判断", "output": "归档", "rule": "超过设定周期的判断移入归档" }
  ]
}
```

## 关键区分：框架 vs 判断

- **框架**（长线，持续迭代）：方法论、信号体系、操作纪律、选股逻辑。人维护，LLM 只读。
- **判断**（短线，会腐烂）：某个板块某周的看多/看空、具体标的分析。LLM 编译+人确认，过期归档。

> 真正有价值的沉淀不是"当时怎么判断的"，而是"框架本身"。判断失效了就归档，框架要持续打磨。

## 编译规则

- **原始资料 → 认知**：LLM 只提取要点和信号，不保留原文。每条判断必须带日期标记。
- **过期判断 → 归档**：超过设定周期（如 2 周）的判断，由 `archive.py` 脚本自动移入归档区。归档不删除，保留作为复盘素材。

## 时效处理

```yaml
# 时效规则示例
判断有效期: 14天
归档触发: 超过有效期自动移动到 {归档}/
框架更新: 手动，LLM 只读不写
```

## Lint 检查项

- 超过有效期未归档的判断
- 框架和归档判断之间有无矛盾
- 原始资料是否有未提取的积压
- 认知区的判断是否都带了日期标记

## 产出模板

| 模板 | 用途 |
|------|------|
| `templates/inbox-item.md` | 灵感库条目（公共） |

> 框架和认知的内容格式由用户自定义，本策略不强制产出模板——因为不同领域（交易/竞品/行业动态）的框架结构差异大，模板化反而限制灵活性。
