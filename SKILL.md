# KB Engine · 知识库引擎

## 这是什么

一套基于 Karpathy LLM Wiki 架构的多策略知识库引擎。核心思想：**策略是数据（模板），不是代码（逻辑）。** 引擎只管流程，策略模板定义规则，目录结构通过 JSON 配置。

## 核心概念

- **五个策略**：不同知识类型用不同维护策略（持续积累/主题研究/时效腐烂/队列追踪/周期运营）
- **策略模板**：每个策略的规则定义（编译规则、时效处理、产出物），是 markdown 文件
- **目录配置**：每个领域的 `_目录配置.json`，定义实际目录路径，改目录只改这个文件
- **脚本**：文件操作和逻辑判断由 Python 脚本执行，LLM 负责理解意图和编译内容

## 四个命令

### 1. 初始化新领域
```
/知识库 初始化 [领域名]
```
读取策略模板 → 生成 `_目录配置.json` → 用户确认 → 创建目录

### 2. 处理新内容
```
/知识库 处理 [文件/内容]
```
判断策略 → 加载策略模板 → LLM 编译 → 脚本写入正确目录

### 3. 迁移目录结构
```
/知识库 迁移 [领域名]
```
diff 新旧 `_目录配置.json` → 生成迁移计划 → 用户确认 → 批量移动+更新链接

### 4. 维护任务
```
/知识库 维护
```
lint 健康检查 → 清理灵感库 → 检测更新 → 归档过期内容

## 工作流

1. **LLM 收到用户指令**，判断调用哪个命令
2. **LLM 读取对应策略模板**（`strategies/`），理解该策略的编译规则
3. **LLM 读取 `_目录配置.json`**，确定文件该放哪、从哪读
4. **LLM 调用脚本**（`scripts/`）执行文件操作（移动/比对/提交）
5. **LLM 按产出模板**（`templates/`）生成内容
6. **脚本执行 git 提交**，记录变更

## 文件结构

```
kb-engine/
├── SKILL.md                    ← 本文件
├── strategies/                 ← 策略模板（规则定义）
├── templates/                  ← 产出模板（填空用）
├── rules/                      ← 公共规则（跨策略共享）
├── config/                     ← 配置文件
├── scripts/                    ← Python 脚本（核心逻辑）
└── lib/                        ← 共享 Python 模块
```

## 重要原则

1. **策略模板用逻辑名引用目录**（如 `{知识卡片}`），不写死路径，实际路径在 `_目录配置.json` 里
2. **LLM 负责理解和编译，脚本负责文件操作**，两者分工
3. **已审核内容靠权限隔离**（`_目录配置.json` 的 `writable` 字段），不靠事后校验
4. **raw 层用 md5 做增量触发，wiki 层用 git 做版本管理**
5. **新增策略只需加一个模板文件 + 在策略索引里加一行**
