# 换季操作清单

## 用途

策略 A（持续积累型）专用的跨阶段操作。每隔一个阶段（如半年到一年），做一次五分钟换季。

## 操作步骤

1. **新建下一阶段的阶段重点文件**
   - 复制模板，填新阶段的里程碑
   - 路径：`{阶段重点}/下一阶段.md`

2. **更新索引**
   - 索引文件的"当前阶段"链接指向新文件

3. **浏览匹配新阶段的知识卡片**
   - 查 `stage` 字段匹配新阶段的卡片
   - 心里有个数，知道接下来要用哪些卡片

4. **标记旧阶段**
   - 旧阶段文件加标记：`> 已度过（YYYY.MM–YYYY.MM）`
   - 不删，保留作为历史

## 注意事项

- **不搬运卡片**：卡片的 `stage` 字段是"激活时间"，到阶段自然进入视野，不需要移动文件
- **不重构结构**：换季只是推进阶段，不是重新设计目录。结构在第一个月定下来后只增不改
- **五分钟内完成**：这不是一个大工程，是轻量的阶段推进

## 自动化

```bash
# 检查是否该换季
python scripts/lint.py --domain 育儿 --check season

# 执行换季（交互式）
python scripts/init_domain.py --domain 育儿 --season-transition
```

---

> **适用范围**：策略 A（持续积累型）
> **执行频率**：按阶段周期，如每半年到一年
